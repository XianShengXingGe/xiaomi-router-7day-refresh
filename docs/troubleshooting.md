# Troubleshooting

## Check router capability

```sh
uname -m
ls -l /dev/net/tun 2>/dev/null
cat /dev/net/tun 2>&1 | head -5
which iptables
which nft
```

Good TUN sign:

```text
cat: read error: File descriptor in bad state
```

This usually means `/dev/net/tun` exists and is usable.

## Check running status

Start with the friendly status command:

```sh
/data/xiaomi-router-autosign-status.sh
```

For lower-level checks:

```sh
ps | grep xiaomi-router-autosign
ip route get 10.7.0.1
iptables -L FORWARD -n -v --line-numbers | head -12
iptables -t nat -L PREROUTING -n -v --line-numbers | head -12
cat /tmp/xiaomi-router-autosign.log
```

Expected high-level results:

```text
[OK] helper process is running
[OK] TUN interface sidestore is up
[OK] route to 10.7.0.1 uses sidestore
```

## `open TUN failed: device or resource busy`

A previous process is still holding the TUN device.

```sh
for PID in $(ps | grep '[s]idestore-vpn-go' | awk '{print $1}'); do
  kill -9 "$PID" 2>/dev/null
done
for PID in $(ps | grep '[x]iaomi-router-autosign' | awk '{print $1}'); do
  kill -9 "$PID" 2>/dev/null
done
ip link del sidestore 2>/dev/null
```

Then restart.

## iPhone IP changed

The firewall rules are bound to the iPhone LAN IP. Fix the iPhone IP in the Xiaomi router's DHCP static IP assignment page, then update `IPHONE` in:

```text
/data/xiaomi-router-autosign.conf
```

Then restart:

```sh
/data/xiaomi-router-autosign-start.sh
```

## SideStore still reports LocalDevVPN/pairing issue

Router-side packet rewriting may already be working. Check whether the counters increase. Then verify on iPhone:

1. SideStore has Local Network permission.
2. No other VPN is active.
3. The pairing file is valid and generated by the expected iLoader/SideStore flow.
4. SideStore is killed and reopened after replacing the pairing file.

## ShellClash rule order

If the NAT `RETURN` rule is not at the top after reboot, make sure the startup script still waits for ShellClash before inserting rules:

```sh
grep '^sleep 60' /data/xiaomi-router-autosign-start.sh
```
